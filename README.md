# Dota Coach AI

Приложение-тренер для Dota 2 на Python + PySide6.

## Что умеет версия 0.9.3

- загружает последние игры игрока через OpenDota;
- переводит Steam64 в Dota account_id автоматически;
- считает winrate, KDA, GPM/XPM, LH/min, урон, вижен и стаки;
- определяет основную роль: Carry, Mid, Offlane, Soft Support, Hard Support;
- добавляет вкладку **Герои**: 126 героев, типы Сила/Ловкость/Интеллект/Универсальность, контрпики, контр-предметы и мини-игра Draft 5v5;
- добавляет вкладку **Предметы**: 200+ предметов, описания, лучшие роли/герои и мини-игра оценки билда;
- сравнивает игрока с ориентиром выбранной роли;
- показывает pro-style референсы по стилю игры;
- строит план улучшений на следующие игры;
- принимает live-снапшоты Dota 2 через Game State Integration на `http://127.0.0.1:3000/`;
- автоматически запускает локальный GSI-сервер при старте приложения;
- устанавливает GSI cfg сразу в несколько возможных папок Dota 2, чтобы убрать проблему «данных пока нет»;
- имеет кнопку **Тест сервера**, которая отправляет тестовый JSON в локальный GSI receiver;
- добавляет вкладку **Pro Lab** с картинками-паттернами по карте, свежими pro-матчами OpenDota, таймкодами и replay/demo загрузкой;
- работает с локальной Ollama-моделью, если она установлена;
- имеет fallback-анализ без Ollama.

## Запуск как обычное приложение на Windows

### Вариант 1 — без консоли и без сборки EXE

1. Распакуй архив в удобную папку, например `C:\Games\DotaCoach`.
2. Один раз запусти файл:

```text
setup_windows.bat
```

Он создаст `.venv`, установит зависимости и добавит ярлык **Dota Coach AI** на рабочий стол.
После этого запускай приложение двойным кликом по ярлыку или по файлу:

```text
DotaCoach.vbs
```

Консоль при обычном запуске открываться не будет.

### Вариант 2 — собрать настоящий EXE

На Windows запусти:

```text
build_windows_exe.bat
```

После сборки готовая программа будет здесь:

```text
dist\Dota Coach AI\Dota Coach AI.exe
```

Этот `.exe` запускается как обычная программа без консоли. Сборку Windows-`.exe` нужно делать именно на Windows.

## Обычный запуск из консоли для разработки

```bash
cd DotaCoach
python -m venv .venv
```

Windows:

```bat
.venv\Scripts\activate
pip install -r requirements.txt
python run.py
```

macOS / Linux:

```bash
source .venv/bin/activate
pip install -r requirements.txt
python run.py
```

## Настройка игрока

По умолчанию в `config.py` стоит Steam64 из исходного проекта. Его можно поменять прямо в верхнем поле интерфейса или через переменную окружения:

```bat
set DOTA_STEAM64=7656119XXXXXXXXXX
```

Можно вставлять и обычный `account_id` OpenDota — приложение само поймет формат.

## Live Coach через Dota 2 GSI

В версии 0.9.3 GSI-сервер запускается автоматически, но вкладка **Live Coach** оставляет кнопки для ручной проверки.

1. Запусти приложение.
2. Открой вкладку **Live Coach**.
3. Нажми **Тест сервера**. Если тест прошел, значит локальный сервер приложения работает.
4. Нажми **Починить Live** или **Установить cfg в Dota 2**. Если Dota стоит нестандартно, нажми **Выбрать cfg-папку** и укажи папку `cfg` вручную. Приложение пишет один и тот же файл сразу в несколько мест:

```text
steamapps\common\dota 2 beta\game\dota\cfg\gamestate_integration_dota_coach.cfg
steamapps\common\dota 2 beta\game\dota\cfg\gamestate_integration\gamestate_integration_dota_coach.cfg
steamapps\common\dota 2 beta\dota\cfg\gamestate_integration_dota_coach.cfg
steamapps\common\dota 2 beta\dota\cfg\gamestate_integration\gamestate_integration_dota_coach.cfg
```

Самый важный путь — прямой файл в `game\dota\cfg`. Старые инструкции часто советуют подпапку `gamestate_integration`, но на части установок Dota читает GSI cfg именно напрямую из `cfg`.

5. В Steam открой **Dota 2 → Свойства → Общие → Параметры запуска** и добавь:

```text
-gamestateintegration
```

6. Полностью закрой Dota 2 и запусти её заново.
7. Зайди в матч/лобби. Во вкладке **Live Coach** должны появиться герой, время, KDA, ресурсы и подсказки.

Если после **Тест сервера** данные из Dota всё равно не идут, причина почти всегда не в приложении, а в одном из пунктов: Dota не перезапущена после установки cfg, cfg лежит не в той папке, параметр запуска не сохранился, порт 3000 занят другим процессом, или Windows Firewall заблокировал локальные HTTP-запросы.

## Pro Lab

Вкладка **Pro Lab** добавляет учебный блок:

- картинки-паттерны по карте: фарм керри, руны мида, вижен саппорта, давление оффлейна;
- свежие pro-матчи через OpenDota `/proMatches`;
- короткие таймкоды/моменты по объектам и дракам, если OpenDota отдал детали матча;
- сравнение твоего последнего матча с pro-матчем по GPM/XPM, LH/min, смертям и tower damage;
- кнопки скачивания `.dem`/`.dem.bz2` replay-файлов для твоего матча и выбранного pro-матча.

Важно: приложение не скачивает видео с YouTube/Twitch и не нарушает авторские права. Оно работает с публичными match/replay данными. Автоматическая нарезка настоящего MP4 из `.dem` требует запуска Dota 2/рендера replay в игровом клиенте, поэтому сейчас приложение скачивает replay и дает таймкоды для сравнения.

## Ollama

Если Ollama не установлена, приложение всё равно работает и показывает встроенный разбор по метрикам.

Чтобы включить локальную модель:

```bash
ollama pull qwen3:8b
```

Можно выбрать другую модель:

```bat
set OLLAMA_MODEL=llama3.1:8b
```

Или отключить Ollama полностью:

```bat
set DOTA_COACH_USE_OLLAMA=0
```

## Важное ограничение

OpenDota дает в основном post-match аналитику. Настоящий live-режим работает только через Game State Integration, когда Dota сама отправляет разрешенные JSON-данные в локальное приложение. Проект не читает память игры, не сканирует экран, не показывает информацию из тумана войны и не автоматизирует действия игрока.

## Структура проекта

```text
DotaCoach/
  DotaCoach.pyw                 # запуск без консоли через pythonw.exe
  DotaCoach.vbs                 # скрытый Windows-лаунчер для .venv
  setup_windows.bat             # первая установка + ярлык на рабочем столе
  build_windows_exe.bat         # сборка Windows exe без консоли
  DotaCoach.spec                # PyInstaller-конфигурация
  run.py                        # запуск приложения из консоли
  coach_ui.py                   # QApplication + стиль
  app_paths.py                  # пути ресурсов, поиск Steam/Dota cfg
  config.py                     # настройки
  dota_api.py                   # OpenDota client + агрегация метрик
  role_detector.py              # определение роли
  pro_players.py                # role benchmarks + pro-style референсы
  pro_lab.py                    # гайды, pro feed, replay download helpers
  dota_catalog.py               # база героев/предметов, draft/build анализ
  ai_analyzer.py                # Ollama/fallback анализ
  live_gsi.py                   # локальный HTTP GSI receiver
  ui/main_window.py             # основной интерфейс
  ui/assets/guides/*.png        # картинки-паттерны карты
  ui/assets/catalog/heroes/*.png # иконки героев
  ui/assets/catalog/items/*.png  # иконки предметов
  ui/assets/dota_coach.ico      # иконка приложения
  config/gamestate_integration_dota_coach.cfg
```
